# 苏小有设计说明

本文档记录苏小有桌面端界面的基本设计原则。具体颜色、字号和组件实现以 `frontend/src/app/globals.css` 与现有组件为准。

## 设计目标

苏小有是办公场景中的 AI 工作助理，界面应当安静、清晰、可靠，适合长时间处理文档、表格、会议材料和本地文件。

视觉风格不追求营销感，而是强调信息密度、可读性和稳定操作。

## 基础原则

- **本地工作感**：界面要像一个桌面工作台，而不是宣传页面。
- **中文优先**：用户可见文案优先中文表达，避免无必要英文。
- **低干扰**：减少装饰性背景、夸张渐变和无意义动画。
- **可扫描**：列表、设置、活动记录和文件面板要便于快速浏览。
- **可控感**：文件修改、命令执行、远程访问等高影响操作要让用户清楚知道发生了什么。

## 颜色

深色主题默认值：

```text
强调色：#339CFF
背景色：#000000
前景色：#FFFFFF
```

颜色应服务于信息层级：

- 主背景保持简洁。
- 强调色用于当前选中、焦点、主要按钮和关键状态。
- 成功、警告、错误状态使用稳定且易识别的语义色。

## 排版

- 桌面端基础字号应保持紧凑，但不能牺牲可读性。
- 长文本、报告、代码块和表格要有明确层级。
- 按钮、标签、侧边栏和状态文案避免过长，必要时使用截断或换行。

## 布局

- 左侧用于项目和会话导航。
- 中间用于对话和主要工作区。
- 右侧用于文件、进度、上下文和活动信息。
- 设置页应以表单和分组为主，避免卡片套卡片。

## 图标与品牌

- 应用图标和状态栏图标应与 `Suxiaoyou-Logo/mascot.png` 的视觉结构一致。
- 状态栏图标应适配 macOS 模板图标规则，保持简洁可辨。
- README 和公开说明中可以使用品牌图，但不要继续使用旧品牌素材。

## 文案

苏小有的表达应简洁、明确、偏工作化。

推荐表达：

- “今天，让苏小有陪你一起完成什么？”
- “生成报告”
- “整理会议纪要”
- “分析表格”
- “写一封汇报邮件”

避免表达：

- 无必要英文按钮或说明。
- 与当前产品无关的开源社区口号、下载徽章、旧项目历史。
- 过度营销、夸张承诺或不明确的能力描述。
