# Changelog

## v1.1.1-rc.1 — 2026-07-24

应用与安装包内部版本为 `1.1.1`，候选发布标签为 `v1.1.1-rc.1`。本候选版继续使用公开、
非 latest 的 `UNSIGNED-DEGRADED` 配置；完整边界见
[`docs/releases/v1.1.1-rc.1.md`](docs/releases/v1.1.1-rc.1.md)。

### 工作区恢复根修复

- 用耐重启的 workspace identity v2 取代 POSIX `st_dev` 持久身份。macOS/APFS 在重启或重新挂载后
  改变 device 编号时，不再把原工作区误判为已被替换并导致本地服务启动失败。
- 旧 `stat-v1` 数据以“建立 durable token、迁移并校验外部版本树、最后更新数据库”的顺序幂等升级；
  缺失、损坏或身份不可证的工作区与 journal 会单独阻断并保留证据，不影响安全工作区和后端就绪。
- checkpoint、rewind、文件版本、事务 journal、实时工具上下文与 Git worktree 统一绑定 durable token，
  同路径重建目录不能继承旧工作区历史。
- 受控 Git worktree 继续隔离用户级 hooks、filters、凭据助手等可执行配置，同时对白名单中的
  `core.autocrlf`、`core.eol`、`core.safecrlf` 做严格规范化并显式传入 Git；Windows 的正常 CRLF
  工作树不再因安全隔离环境采用不同换行策略而被误报为脏仓库。

### 六目标与八个安装包

- 新增原生 Windows ARM64 NSIS，与 Windows x64 分别构建和执行安装、启动、退出及无孤儿进程验证。
- 发布矩阵现在覆盖 Windows x64/ARM64、macOS x64/ARM64、Linux x64/ARM64 六个目标；Linux 每个架构
  同时提供 DEB 与 RPM，因此 Release 共包含八个安装包。
- release manifest、CHECKSUMS 与 native package lifecycle evidence 要求八个安装包精确齐全，禁止
  使用 x64 证据代替 Windows ARM64 或复用另一目标的生命周期报告。

### 信任边界

- Windows 安装包没有 Authenticode 发行方签名；macOS 应用仅为 ad-hoc 签名且 DMG 未做 Developer ID
  签名、公证或 staple；Linux DEB/RPM 未做包或仓库签名。
- Cloudflare 官方当前没有 Windows ARM64 原生 `cloudflared` 资产；ARM64 包不会用 x64 下载冒充，
  未自行提供兼容隧道二进制时 Quick Tunnel 会失败关闭，本地功能仍可使用。
- 前端升级到 Next.js `15.5.21`，并锁定已修复的 DOMPurify `3.4.12`、PostCSS `8.5.22` 与
  Sharp `0.35.3`，使生产依赖审计恢复为零已知漏洞。
- 冻结 authoritative Office renderer、真实集成和受控 Beta 证据仍不包含在公开包中；文件名和机器可读
  清单持续显式标注 `UNSIGNED-DEGRADED`。

## v1.1.0 — 2026-07-18

本版本以公开、非 latest 的 `UNSIGNED-DEGRADED` 预发布配置交付。七个安装包全部带有明确的降级文件名；
发布不宣称已具备发行方签名、Apple 公证、冻结 authoritative Office renderer、真实集成或受控 Beta 证据。
完整边界见 [`docs/releases/v1.1.0.md`](docs/releases/v1.1.0.md) 和随 Release 提供的机器可读清单。

### Runtime、ACP 与恢复

- 新增统一运行时事件、持久 turn checkpoint、mutation ledger 与 workspace instance；受控文件变更可以
  预检并 rewind，冲突时 fail-closed，外部不可逆副作用保持显式可见。
- 新增 stdio ACP Beta、受控 detached Git worktree、来源受限 Hooks，以及独立只读验证 Agent；权限、
  审计、急停、取消和预算继续由同一安全边界约束。
- 桌面端加入 runtime/checkpoint 控制面、Hook 撤销与来源状态、Validator 结果和用户 Office 模板管理。

### Office 编辑与视觉闭环

- 扩展 DOCX、XLSX、PPTX 的模板、图表和复杂格式安全子集，加入签名第一方模板与受审批的用户模板导入。
- 新增 checkpoint-bound 预览、结构与视觉差异、只读验证、最多两轮修复和 rewind 联动；Office 提交继续
  经过事务 seal、独立重开、确定性检查和原子提交。
- 公开包不包含项目冻结的 authoritative renderer。权威 Office authoring/视觉提交在 renderer 缺失时
  保持不可用；仅当主机存在兼容 LibreOffice 时提供明确标为 approximate 的预览。

### 发布信任边界

- Windows 与 Linux 安装包未做发行方签名；macOS 应用仅为 ad-hoc 签名，DMG 未做 Developer ID
  签名、Apple 公证或 staple，Gatekeeper 可能阻止直接启动。
- Release 为 prerelease、非 latest。七个安装包、校验和、release manifest、Office restricted
  compatibility 和 degraded capabilities 共同绑定同一 `v1.1.0` tag 与 commit。

## v1.0.0-rc.1 — 2026-07-15

本版本聚焦“安全可交付的本地办公 Agent”，不以复制 WorkBuddy 的生态、团队协作或云端运行时为目标。
该标签是 `v1.0.0-rc.1` 候选版，应用与安装包内部版本保持 `1.0.0`；在七个原生安装包、真实服务
E2E、签名/公证和受控 Beta 门禁全部完成前，本条目不代表稳定版已经发布。

### 办公文件与恢复

- 新增受限声明式 DOCX、XLSX、PPTX 创建和有限编辑工具；保持工作区、格式、大小和宏/外部对象边界，
  写入通过临时文件、重开验证和原子替换完成，不为 Office 操作开放任意 Shell 或 Python。
- `write`、`edit`、`apply_patch` 和 Office 在各桌面平台通过受守卫的声明目标流程覆盖已有文件，并创建
  应用私有、内容寻址且带 SHA-256 的版本。Linux Bash/Python 使用 Bubblewrap，macOS 使用 Seatbelt，
  命令先在私有副本运行，只有成功退出才提交，超时、取消和非零退出不暴露部分写入；Windows 命令
  使用 Job Object 回收进程树但直接作用于已批准工作区，不提供文件系统或网络隔离，也不保证失败回滚。
  多文件 POSIX 提交还会持久化恢复日志，后端若在提交中途退出，会在下次启动、任何 Agent 写入前
  校验并回滚。提交与恢复通过 workspace/parent dirfd、原子交换和 no-replace rename 抵御并发路径替换，
  回滚只处理仍匹配本事务输出的对象；hard-link 变更在 v1 fail-closed 拒绝。支持按工作区列出、校验并
  原子恢复，恢复前也会保留被替换内容。任何曾作为可见目标的 inode 都保留为可定位的隐藏 recovery
  sidecar，避免旧 open FD 在 finalize 后写入不可达；v1 不自动回收这些文件，成功结果通过 metadata
  返回路径。空目录回滚采用预制目录加 no-replace 安装，同名用户重建项只报冲突且绝不 chmod。
- 声明式工具使用只覆盖已声明目标与 Office 本地图片依赖的稀疏事务视图；无关的超大文件或特殊文件
  不再阻断单文件操作，同时保留只读输入前后 CAS、精确写集合和多文件失败回滚。没有活动工作区时
  write/edit 明确零副作用拒绝，不恢复旧版任意绝对路径写入。
- Office 编辑对未知 OOXML 部件、关系、custom XML 和潜在静默丢失 fail-closed；本地图片只作为事务
  的只读输入，不会被当作输出发布。
- 读取图片在 Base64 编码前限制为 20 MiB 并执行 fd/no-follow 身份校验；目录列表以内存有界方式
  返回全局稳定排序的前 1000 项及截断元数据。

### 外部能力与安全中心

- 新增腾讯文档官方 MCP 连接器，固定官方端点与原始 `Authorization` Token 鉴权；Token 进入凭据存储，
  工具列表和调用采用双重允许列表，写操作遵循权限确认，危险删除默认关闭。
- 新增单一 SiliconFlow 文生图工具；每次付费调用都要求不可覆盖的用户确认，网络发送前写入持久计费
  operation ledger，同一 call ID 不会重复提交；取消、超时或下载失败会标记为计费状态不确定并禁止
  自动重试。产物执行完整 PNG 重开校验和工作区内无覆盖保存，不向模型暴露 API Key 或临时下载 URL。
- 新增 Security Center Lite：展示来源与能力、凭据是否配置、外部工具开关、持久脱敏审计和急停。
  急停会中止当前生成，并停止调度器、连接器及由应用托管的外部运行时；重启后仍保持停止状态。
- 新增 `0006_v100_security_audit` 与 `0007_v100_invocation_source` 数据库边界，持久区分工具来源和
  desktop/scheduler/channel/OpenAI-compatible 根调用来源，并将应用、前端、后端和桌面安装包元数据
  同步为 `1.0.0`。
- 本地 MCP 启动审批绑定最终可执行文件路径、字节 SHA-256、参数、环境和工作目录；审批存储、PATH
  变化、并发启停和急停均 fail-closed。自动化继续保持无人值守只读，循环模板同步改为审阅、拟稿和
  建议，不再承诺会被安全策略拒绝的归档、直接改文档或删除数据。

### 跨多轮目标模式

- 新增本地桌面 `/目标` 与 `/goal`：可创建、查看、编辑、暂停、继续和清除一个会话级持久目标，
  并在 Goal 卡片中展示状态、运行轮数、活跃时间、Token 预算、最近阻塞和完成摘要。
- 新增 `0008_v100_session_goal` 与 `0009_v100_goal_usage_ledger` 迁移，持久保存 Goal、逐切片
  GoalRun、Goal 关联 Todo 和幂等用量来源；并发编辑/恢复使用 revision CAS，崩溃或重启不会重放
  结果不确定的副作用。
- 自主续跑以顶层切片公平调度，已提交的真实用户输入优先；Provider 与工具准入都受安全暂停、急停、
  Token/模型费用/活跃时间/轮数预算、连续错误和无进展断路器约束。子任务用量计入父 Goal。
- Goal 保存创建时的服务端权限上限，并与当前 Session、全局和 Agent 策略取更严格交集；当前策略
  只能收紧，后续放宽不会给旧 Goal 扩权。完成必须由受限 Goal 工具提交证据并通过服务端校验。
- Goal 的创建目标、阶段进展和完成摘要作为普通对话消息持久显示；修复目标对话中旧问答被隐藏的问题，
  并移除“目标累计 TOKEN 分解”和“目标累计上下文用量（含缓存读取）”展示。
- `GOALS_RELEASED` 与 `AUTONOMOUS_GOALS_RELEASED` 在源码中默认开启，但发布范围仅限受本地会话
  保护的单一桌面客户端。Remote/消息渠道、移动新建、多窗口实时广播和退出后云端运行未开放；
  其他窗口需刷新或重新聚焦后从数据库收敛。第三方付费工具费用不并入模型费用预算，仍逐次确认。

### 对话与凭据体验

- 修复侧栏对话右键菜单点击“重命名”后输入框一闪而过的问题；右键菜单与标题右侧菜单现在共用稳定的
  行内重命名生命周期。
- 多轮对话导航的未悬停短横统一保持轻量灰色；悬停时仅加粗、略微变长。移除总轮数按钮、可见的
  “第 x 轮”文字和重复弹层，避免遮挡正文。
- 模型凭据改为按需激活并聚合保存，修复环境变量凭据引用无法解析以及同一进程内重复触发系统凭据
  请求的问题；macOS 覆盖升级的 Keychain 提示次数仍作为 RC 公开前的人工阻断验收。

### 构建与验证

- 本地后端构建改为 fail-closed 包装器：macOS 在 PyInstaller 前后检查全部 Mach-O 的部署目标，
  固定 `--clean` 和声明的 `MACOSX_DEPLOYMENT_TARGET`；可显式选择 uv 托管 Python，避免把仅兼容
  构建机新系统的 Homebrew Python/OpenSSL 带入安装包。
- 当前工作树已使用 uv 托管 Python 3.12.13 和 PyInstaller 6.21.0 重建 Apple Silicon 本地未签名
  DMG，并完成 150 个 Mach-O 架构/最低系统版本、49 项许可资源、冻结后端/Office smoke、只读挂载、
  拷贝安装和桌面/后端生命周期验收。该产物来自未提交工作树，只是本机预检，仍不替代七平台同 tag
  构建、Developer ID 签名、公证或正式 RC 证据。

### RC 草稿与公开发布前置条件

- RC 草稿必须包含同一 tag/commit 的七个原生安装包和四份证据文件；五个平台的安装、启动、退出、
  Office/文件事务及无孤儿进程门禁必须全部通过，且不存在 P0/P1 或已知数据损坏。
- Windows 声明式单目标文件/Office 写入与版本恢复必须在原生 runner 通过；Windows Bash/Python 的
  已批准工作区直接执行限制必须如实披露，不能写成具备 POSIX 私有事务、文件系统或网络隔离。
- 公开 RC 前还需完成人工 macOS 覆盖升级验收：保留旧凭据安装、首次实际请求至多一次系统密码提示，
  后续请求及重启不重复并能成功调用；同时完成至少 20 名受控用户、200 个核心任务、连续 7 天且零
  数据丢失的 Beta 记录和三道计分卡验证。缺少这些证据时 Release 保持草稿。
- 稳定版还需腾讯文档真实账号 E2E、SiliconFlow 真实计费账号 E2E，以及 macOS Developer ID 签名、
  Apple 公证和 staple；mock 合同测试不能替代真实服务证据。

## v0.9.0-rc.1 — 2026-07-14

本版本把命令执行、权限、凭据、数据库升级和发布产物收敛到可验证的安全边界。该标签是
`v0.9.0-rc.1` 候选版；应用与安装包内部版本保持 `0.9.0`。

### 执行与权限边界

- Python 与 Shell 工具在 Linux 改由 Bubblewrap OS 级沙箱托管；Linux DEB/RPM 将
  `bubblewrap` 声明为安装依赖，缺失时拒绝执行。
- macOS 与 Windows RC 暂不开放 Bash/Python 命令执行。macOS Seatbelt 能限制资源，但没有
  可靠、公共且无需额外授权的完整后代进程回收边界；Windows Job Object 能回收进程树但不
  提供文件系统/网络隔离。因此两端都明确 fail-closed，不退回无隔离执行。
- 非交互任务遇到 `ask` 权限必须失败；子任务继承父任务的 workspace、权限模式和已批准
  边界，不能自行扩大路径或执行权限。
- 新安装默认进入 Ask 模式，首次引导必须先选择工作区；没有有效工作区的旧配置也会重新
  进入该安全边界。

### Provider、凭据与数据恢复

- Anthropic 与 Google Gemini 改为官方生产 SDK，并加入原生模型目录、消息/工具映射、流式
  事件回归及最终冻结后端的离线构造 smoke；移除不存在的 `yakagent` 运行时假设。由于当前
  消息历史还不能保留 signed thinking/thought signature，本候选版不宣称这两个原生适配器
  支持 reasoning；适配器会忽略遗留的 reasoning 偏好且不向 SDK 发送 thinking 配置。
- Provider、OAuth、MCP 与渠道凭据迁移为不透明引用，优先存入 macOS Keychain、Windows
  Credential Manager 或 Linux Secret Service；无可用原生服务时仅使用原子写入的私有
  `0600` 文件回退。
- 新增正式 `0005` Alembic 版本边界，以及带 SHA-256 清单、SQLite `quick_check`、原子替换
  和恢复前安全备份的离线列举/恢复命令；不在正式数据库上原地降级。
- Remote 与消息 Channels 的高权限 Agent 路径在本候选版由代码级发布开关关闭，旧环境变量
  不能重新开启其路由、隧道或消费者。

### 构建与发布门禁

- CI 增加 Python、三份 JavaScript 生产依赖图与 Rust 漏洞审计，并执行 production frontend、
  冻结后端、Rust release build、Playwright 核心场景和真实 Linux DEB 安装事务。
- 最终后端 bundle smoke 同时验证官方 Anthropic/Gemini SDK 可构造；Linux 产物必须证明
  沙箱的文件系统、网络、环境变量和后代进程隔离，macOS/Windows 产物必须证明命令执行
  保持 fail-closed。
- macOS Intel 继续在原生 Intel runner 重建后端与桌面应用，并对最终 `.app`、DMG、Mach-O
  架构、最低系统版本、安装后启动/退出和孤儿进程执行门禁；不使用 Apple Silicon 交叉产物
  代替 Intel 验证。

### 已知限制

- 只有 Linux 在 rc.1 开放命令执行；macOS 与 Windows 均不可用。如果 v0.9.0 的完成定义要求
  三平台命令能力对齐，则该项尚未成熟：macOS 需同时解决沙箱后代的强制回收，Windows 需补
  AppContainer/受限文件系统与网络边界。
- 候选版发布以同一 tag 的七个原生安装包全部通过 runner 验证为前提；仅有工作流配置或本地
  预检不构成已通过证据。发布页见 [`v0.9.0-rc.1`](https://github.com/keliihall/suxiaoyou/releases/tag/v0.9.0-rc.1)。
- macOS 没有 Apple Developer ID 与公证凭据时，RC 仍只能发布明确标注的测试包：DMG 本身
  未签名、未公证，内含的 `.app` 仅使用 ad-hoc 临时签名。

## v0.8.3 — 2026-07-13

本版本补齐大文件原生操作、PPTX 静态预览与长对话历史定位，并继续保持本地桌面能力的会话与路径授权边界。

### 本地化与品牌

- 修复英文模式下设置、首次引导、启动恢复、移动端、活动记录和错误提示仍显示中文的问题；英文展示名统一为 `suyo`，中文展示名保持“苏小有”。
- 原生应用菜单、系统托盘、窗口标题和 macOS DMG 启动提示会随界面语言在英文与简体中文之间切换；Windows 安装器、控制面板名称和快捷方式支持 English 与简体中文，同时保留既有卸载键与安装目录以支持覆盖升级；Linux 桌面入口提供本地化显示名。
- 后端工具标题、动态结果、服务商名称和可展示错误接收请求语言，不再把中文持久化文案直接泄漏到新的英文会话；历史活动标题由前端按当前语言兼容渲染。
- 运行中排队的每条后续输入分别保存提交时语言；中英文客户端交错排队或安全边界引导时，不再由最后一次请求覆盖整个任务的动态文案语言，并通过 `0004` 数据库迁移兼容 rc.1 数据。
- 公开安装包采用中性 `suyo-*` 文件名；内部包名、进程名、深链接、配置键和 `com.suxiaoyou.desktop` 标识保持不变，避免破坏既有数据与集成。
- 新增英文资源禁中文字符、翻译键完整性、双语品牌、原生菜单与英文关键页面巡航门禁，防止后续版本再次出现单向本地化。

### 文件操作与桌面集成

- “另存为”改为原生分块流式复制：源文件经会话工作区授权，目标由系统保存面板选择，并通过同目录临时文件、`fsync` 与原子替换完成，不再让完整大文件经过 WebView base64/IPC 内存链路。
- 文件菜单新增“选择其他应用打开”；macOS、Windows 与 Linux 分别调用系统原生选择器或受控系统接口，同时保留默认应用打开和文件管理器定位。
- 新增的原生文件操作仅接受本地桌面凭据与当前会话授权文件；远程令牌、未知会话、越界路径和符号链接逃逸均会被拒绝。

### 演示文稿预览

- PPTX 可在应用内离线显示缩略图和单页静态预览，并支持翻页、键盘导航、页码和缩放。
- 预览覆盖文本、图片、表格和基础图形；不执行宏、脚本、外部关系或嵌入对象，不支持的内容会显示降级提示并保留下载和外部应用打开。
- 后端对文件体积、ZIP 条目、解压总量、压缩比、页数、形状、图片像素、文本量和解析时长设置明确上限，避免恶意演示文稿消耗无限资源。

### 长对话导航

- 四轮及以上的对话新增 Codex 风格左侧轮次轨道，可查看用户问题摘要、时间和附件名，并通过鼠标或键盘定位任意历史轮次；窄屏折叠为“对话大纲”。
- 轻量轮次索引覆盖尚未加载的早期消息；定位时只加载目标页及相邻页，不需要把 200 轮完整正文一次性载入前端。
- 历史窗口与实时最新页缓存隔离；新消息到达时保持当前阅读位置并显示未读数，返回最新后恢复真实实时页面，编辑重发也会同步清理失效刻度。

### 已知限制

- 原生文件选择器、文件管理器和“打开方式”对话框的系统按钮由操作系统提供，继续跟随操作系统语言；应用自有标题和提示跟随应用语言。
- PPTX 预览为静态近似渲染，不支持动画、切换效果、视频、完整 SmartArt 或与 Microsoft PowerPoint 完全一致的字体排版；旧 `.ppt` 仍需外部应用或转换。
- 超长对话的轮次索引开销会随消息元数据和用户输入部分线性增长；当前使用分批查询并已覆盖 200 轮回归。
- macOS 候选安装包在没有 Apple Developer ID 和公证凭据时继续使用醒目标注的 ad-hoc 签名，且未公证。

## v0.8.0 — 2026-07-11

本版本以稳定性、长任务可控性和文件体验为主，不进行大范围视觉重构。

### 长任务与交互

- 任务运行期间输入框保持可用，后续文字和附件可按 FIFO 排队，或选择在模型/工具批次结束后的安全边界引导当前任务。
- 待处理输入持久化到数据库，刷新、切换会话或重启后仍可查看；停止、异常或不确定中断后会暂停为 `blocked`，只允许用户显式取消或放回输入框重新编辑，不自动重放副作用。
- 首次请求和排队请求增加持久幂等键，网络响应丢失后的重试不会重复创建会话、执行工具或产生双份输出。
- 权限确认、问题回答和计划审批在继续执行前先持久化决定、来源和时间，再显示“正在提交、已确认、正在继续”回执；重复回答幂等，冲突、过期和不存在的交互返回明确错误。
- SSE 心跳与业务进展分离；非等待用户状态连续 60 秒没有业务事件时提示“任务可能卡住”，并提供重新连接和停止操作。
- 运行时长改为本地化的秒、分秒、小时分秒，例如 `2862s` 显示为 `47分钟42秒`。
- Stop 只有在后端确认后才结束前端运行状态；Shell 停止、超时和应用退出会回收完整子进程树，Windows 使用 Job Object 防止父 shell 先退出后遗留子进程。

### 数据、附件与恢复

- 建立正式 Alembic 历史；真实 v0.7.3 数据库在升级前执行 `quick_check` 和 SQLite online backup，在隔离副本完成迁移后原子替换，失败时保留原库、备份和诊断副本。
- 无项目会话使用稳定的会话托管工作区；上传文件只作为输入快照，输出统一写入系统“文稿/文档/苏小有/生成文件”目录，不再写回上传文件所在文件夹，也不会继承上一个项目目录。
- 上传改为分块、限额、`fsync` 后原子安装；相同内容复用一个物理文件，引用感知回收不会因删除一个会话破坏其他会话或待处理输入。
- `write`、`edit`、`apply_patch`、计划和 artifact 文本落盘使用同目录临时文件与原子替换，磁盘或替换失败不会先截断原文件。
- 重启时将遗留的运行中工具卡、请求和自动化记录恢复为明确的 interrupted/error 状态，避免界面永久停在“运行中”。
- 修复自动化漏跑顺序、并发上限、循环失败误报成功以及关机恢复；漏跑任务在启动时最多补执行一次。

### 文件与导航体验

- 生成文件卡同时提供可见菜单和右键菜单；本地桌面支持预览、默认应用打开、在 Finder/文件资源管理器中显示、复制路径和原子“另存为”，远程端仅保留预览与下载。
- 文件预览增加文本/二进制大小上限和明确错误；本地系统操作、原生选择器和脚本标题转义均增加来源与输入校验。
- “新建项目”和“新对话”成为稳定一级入口，不再依赖项目或会话列表是否存在；覆盖 Windows 仅有普通对话、空列表、搜索无结果、窄窗口和折叠侧边栏。
- 设置“关于”页增加固定的官方 Releases 入口，只用于查看最新版本和手动下载，不静默下载或安装。

### 发布与验证

- PR 门禁覆盖完整后端、前端 unit/type/lint、Rust test/clippy 和 Chromium 核心交互回归。
- Release 门禁在原生 runner 构建并真实安装 Windows NSIS、macOS arm64/x64 DMG、Linux DEB/RPM，启动桌面应用、等待内置后端 ready、执行正常退出并验证无孤儿进程。
- 正式 Release 统一生成五个稳定 ASCII 安装包名、`CHECKSUMS.md` 和只读 `release-manifest.json`；macOS 正式 tag 强制 Developer ID 签名与 Apple 公证，Windows/Linux 未签名状态在发布说明中明确披露。
- `v0.8.0-rc.N` 候选版允许在没有 Apple 凭据时发布 macOS ad-hoc 构建，但应用、DMG 文件名和 Release 说明必须醒目标注未使用 Developer ID、未公证；正式 `v0.8.0` 的签名与公证门禁不会因此放宽。

### 已知限制

- “打开方式”当前调用系统默认应用；枚举并选择任意已安装应用留待后续版本。
- `apply_patch` 保证每个文件独立原子替换，但多文件补丁不是跨文件事务，后续文件失败不会自动回滚先前已完成的文件。
- 上传孤儿文件在应用启动维护阶段回收，长时间不重启时清理可能延后。
- 本地大文件“另存为”仍会经过 base64、JavaScript 数组和桌面 IPC，单次内容限制为 50 MB；后续版本改为原生路径复制或流式传输以降低内存峰值。
- v0.8.0 提供可信版本发现和手动下载入口，不包含应用内自动安装更新。
- 候选版与正式版的安装包内部版本同为 `0.8.0`；从 RC 切换到正式版时，部分系统或包管理器可能要求先卸载 RC 或显式覆盖安装。
