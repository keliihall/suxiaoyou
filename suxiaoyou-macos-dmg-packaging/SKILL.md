# 苏小有桌面端打包

## Scope

Run commands from the 苏小有 repo root unless a step says otherwise.

Keep user edits and the requested version intact. Only change version fields when the user explicitly asks for a version bump. Treat local macOS builds and official full-platform releases as different paths:

- Local Apple Silicon build: produce and verify `苏小有_<version>_aarch64.dmg`.
- Official full-platform release: use `.github/workflows/release.yml` on tagged commits with signing/notarization secrets.

## Target Matrix

Release workflow outputs:

- Windows x64: `desktop-tauri/src-tauri/target/release/bundle/nsis/*.exe`
- macOS arm64: `desktop-tauri/src-tauri/target/aarch64-apple-darwin/release/bundle/dmg/苏小有_<version>_aarch64.dmg`
- macOS x64: `desktop-tauri/src-tauri/target/x86_64-apple-darwin/release/bundle/dmg/苏小有_<version>_x64.dmg`
- Linux x64: `desktop-tauri/src-tauri/target/release/bundle/deb/*.deb` and `bundle/rpm/*.rpm`
- Linux arm64: `desktop-tauri/src-tauri/target/release/bundle/deb/*.deb` and `bundle/rpm/*.rpm` on the native ARM64 runner

Do not claim all-platform local packaging from macOS. Windows and Linux installers need their native CI runners. macOS x64 also needs the x86_64 Rust target plus matching backend and Node runtime.

## Local macOS arm64 Build

1. Confirm version metadata.

```bash
node -e 'for (const p of ["package.json","frontend/package.json","desktop-tauri/src-tauri/tauri.conf.json"]) { const j=require("./"+p); console.log(p, j.version, j.productName || j.name || ""); }'
rg -n '^version = ' desktop-tauri/src-tauri/Cargo.toml
```

2. Check signing availability.

```bash
security find-identity -v -p codesigning
```

If the configured Developer ID identity is missing, use `--no-sign` and clearly report that the app is ad-hoc signed/unsigned for local testing only.

3. Build frontend and backend.

```bash
npm run sync:desktop-meta
npm run build:frontend
npm run build:backend
node scripts/verify-bundle.mjs backend/dist/suxiaoyou-backend
```

If the bundle smoke test times out once during first startup, manually run the backend and probe `/health` plus `/m`, then rerun `verify-bundle`.

4. Build the Tauri app and DMG.

```bash
cd desktop-tauri
npm exec tauri build -- --target aarch64-apple-darwin --config src-tauri/build.macos-aarch64.json --no-sign
```

Omit `--no-sign` only when the local keychain has the Developer ID identity and the user wants a signed package.

5. Return to the repo root, copy the installer to the convenient output directory, and generate checksums.

```bash
mkdir -p ../outputs/installers/v<version>
cp desktop-tauri/src-tauri/target/aarch64-apple-darwin/release/bundle/dmg/苏小有_<version>_aarch64.dmg ../outputs/installers/v<version>/
node scripts/generate-checksums.mjs ../outputs/installers/v<version> ../outputs/installers/v<version>/CHECKSUMS.md
```

Adjust `../outputs` if the repo is not nested under the project folder used in this workspace.

## Full Release via CI

Use this when the user needs every official installer, notarization, or Windows/Linux packages:

1. Ensure the release commit is clean and tagged as `v<version>`.
2. Push the tag so `.github/workflows/release.yml` runs.
3. Confirm workflow jobs pass: `build-windows`, `build-macos`, `build-linux`, and `publish`.
4. Download artifacts from the draft GitHub release or the workflow run.
5. Verify `CHECKSUMS.md` and the manual-download `release-manifest.json` were generated by the publish job.

Stable macOS tags require certificate secrets `APPLE_CERTIFICATE` and
`APPLE_CERTIFICATE_PASSWORD`, plus notarization secrets `APPLE_ID`,
`APPLE_PASSWORD`, and `APPLE_TEAM_ID`. RC tags may use explicitly marked
ad-hoc, non-notarized DMGs. The release contract is manual-download only and
does not produce updater archives, signatures, or `latest.json`.

## Verification

For the generated macOS DMG:

```bash
DMG="desktop-tauri/src-tauri/target/aarch64-apple-darwin/release/bundle/dmg/苏小有_<version>_aarch64.dmg"
APP="desktop-tauri/src-tauri/target/aarch64-apple-darwin/release/bundle/macos/苏小有.app"

ls -lh "$DMG"
shasum -a 256 "$DMG"
plutil -extract CFBundleShortVersionString raw "$APP/Contents/Info.plist"
plutil -extract CFBundleName raw "$APP/Contents/Info.plist"
plutil -extract CFBundleIdentifier raw "$APP/Contents/Info.plist"
file "$APP/Contents/MacOS/suxiaoyou-desktop"
VERIFY_BUNDLE_SKIP_SMOKE=1 node scripts/verify-bundle.mjs "$APP/Contents/Resources/backend"
hdiutil verify "$DMG"
codesign -dv --verbose=4 "$APP" 2>&1 | sed -n '1,80p'
```

Expected local arm64 results:

- version matches the requested version
- name is `苏小有`
- bundle identifier is `com.suxiaoyou.desktop`
- executable is `Mach-O 64-bit executable arm64`
- `hdiutil verify` reports the checksum is valid
- signing is Developer ID for official packages or ad-hoc/unsigned when `--no-sign` was used

## Troubleshooting

Stale venv after moving/renaming the repo:

```bash
head -n 1 backend/venv/bin/pyinstaller
backend/venv/bin/python -m pip install --force-reinstall pyinstaller
```

Stale Cargo/Tauri target cache after moving/renaming the repo:

```bash
rg -n "/old/repo/path" desktop-tauri/src-tauri --glob '!target/**'
rg -n "/old/repo/path" desktop-tauri/src-tauri/target --glob '*.d' --glob '*.json' --glob '*.toml'
cd desktop-tauri/src-tauri && cargo clean
```

Missing signing identity:

```bash
security find-identity -v -p codesigning
```

If it reports `0 valid identities found`, build local test packages with `--no-sign`. Do not present them as notarized release installers.

Node runtime download fails with `UnicodeEncodeError: 'latin-1' codec can't encode characters`:

- Check `backend/scripts/download_node.py` for HTTP headers containing `苏小有` or other non-ASCII text.
- Python's `urllib` encodes request headers as latin-1. Use an ASCII `User-Agent`, such as `suxiaoyou-builder/1.0`.
- In CI, run the script from `backend` with the venv active so `certifi` is available:

```bash
cd backend
source venv/bin/activate
python scripts/download_node.py --platform Darwin-arm64
```

Draft GitHub release asset filenames drop the Chinese prefix:

- Check `gh release view v<tag> --json assets`; `label` may contain the full display name while `name` starts with `_` or `-`.
- After `gh release download`, rename local files to their labels before generating the local `CHECKSUMS.md`.
- Keep the release as draft unless the user explicitly asks to publish it publicly.
