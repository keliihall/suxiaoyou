# 消息由片段序列组成，而不是普通字符串

苏小有中的消息不是带角色标记的一段字符串，而是一组有序的 **片段**。片段类型包括文本、思考、工具调用、步骤完成、压缩、子任务和文件等。

片段类型以可区分联合的形式保存为 JSON，后端定义在 `backend/app/schemas/message.py`，前端在 `frontend/src/components/parts/` 中为不同片段提供对应渲染组件。

这样设计的原因：

- 代理执行不是普通聊天，一条助手消息经常包含多次工具调用、思考块、子任务结果和最终总结。
- 如果压平成字符串，会丢失工具生命周期、可折叠思考、右侧文件面板和可恢复流式输出所需的状态。
- 上下文压缩和消息渠道转发都以片段为边界处理，保持片段结构可以让这些流程更稳定。
