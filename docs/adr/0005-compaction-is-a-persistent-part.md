# 上下文压缩作为持久片段保存

当会话接近模型上下文窗口上限时，系统会向消息流中插入一个 `compaction` 片段，用于记录被总结的内容和被移出模型上下文的内容。相关逻辑位于 `backend/app/session/compaction.py` 和 `backend/app/schemas/message.py`。

压缩前的历史消息仍保存在数据库中，后续发送给模型的是压缩摘要。

这样设计的原因：

- 静默裁剪上下文会让用户和调试者感觉会话突然“忘了东西”。
- 将压缩显式保存为片段后，界面可以展示压缩发生过。
- 被移出的消息仍可从存储中恢复。
- 消息渠道转发可以把压缩视为普通片段类型，而不是特殊分支。
- 回放和导出结果更确定。

代价是数据库占用略有增加。考虑到苏小有以本地存储为主，这个代价可以接受。
