# 苏小有 v0.8.2 发布计划

状态：`v0.8.2-rc.1` 候选发布
目标版本：`0.8.2`
发布主题：跨平台功能对齐、文件预览一致性与轻量交互修复

## 当前发布决策

- 应用和安装包内部版本统一为 `0.8.2`，候选发布标签使用 `v0.8.2-rc.1`。
- Windows x64、macOS arm64/x64 与 Linux x64/ARM64 DEB/RPM 必须从同一提交、同一前后端源码和相同功能门禁构建。
- Windows NSIS 必须使用简体中文，并在原生 runner 完成安装、启动、退出与卸载相关验证。
- 缺少 Apple Developer ID 与公证凭据时，macOS RC 只允许生成明确标注的 ad-hoc、未公证测试包；正式稳定标签仍保持签名与公证 fail-closed。
- PPTX 内置渲染和旧 Office 格式转换继续独立验证，不阻塞本次以稳定性为目标的候选版。

## P0 发布阻断项

### 1. 三平台生成文件能力对齐

- 可见菜单和右键菜单使用同一组操作来源。
- macOS 提供 Finder 定位，Windows 提供文件资源管理器定位，Linux 提供系统文件管理器定位。
- 本地桌面提供预览、默认应用打开、定位、复制路径和另存为；远程端仅保留安全的预览与下载。
- 旧远程连接配置不得让桌面端误判为远程模式或隐藏本地能力。

验收：同一生成文件在三平台菜单中具有相同能力，仅平台名称不同；真实 Windows WebView2 包中右键和可见菜单均可操作。

### 2. Windows 安装器汉化

- NSIS 固定使用简体中文语言资源。
- 自定义安装 hook 不出现英文进度文案。
- 安装、覆盖升级和卸载页面均以中文显示；Windows 或 WebView2 自身系统弹窗不冒充应用内可控文案。

验收：原生 Windows runner 构建并安装成功，人工或截图复核可见页面，安装后应用生命周期门禁通过。

### 3. 任务时间降噪与一致性

- 当天显示刚刚、分钟或小时；前一、前二自然日显示昨天、前天；三日以上显示日期，跨年显示年份。
- 运行中状态替代右侧时间；按时间分组时不重复展示无意义的行级日期。
- 排序、分组和显示统一使用用户选择的创建时间或更新时间。
- 中文、英文、上海时区自然日边界和 Windows 缩放均有行为回归。

### 4. 设置返回应用

- 返回控件整行可点击，最小高度 40px，有明确 hover 和键盘焦点状态。
- macOS 窗口拖拽区与交互热区分离，Windows/Linux 标题栏不受影响。
- 优先返回进入设置前的非设置页面；直接打开设置时回退新对话。

### 5. 文件预览入口一致性

- 生成文件卡、工作区文件和对话文件入口使用同一磁盘读取路径，不把空内容交给内容型 renderer。
- 常见浏览器可解码的 PNG、JPEG、GIF、WebP 和 BMP 图片在本地面板预览，并在卸载时释放 Blob URL。
- 二进制读取继续受 50 MiB 上限和工作区授权约束。

## 发布门禁

- 前端 typecheck、lint、unit 与核心 Playwright 全部通过。
- 后端完整 pytest、生产依赖审计和 bundle smoke 全部通过。
- Rust fmt、test、clippy 与桌面生命周期测试全部通过。
- Windows NSIS、macOS arm64/x64 DMG、Linux x64/ARM64 DEB/RPM 均通过原生 runner 的安装包完整性与启动退出验证。
- Publish job 只接收七个完整安装包，生成 `CHECKSUMS.md` 与只读 `release-manifest.json` 后准备草稿 pre-release；人工下载复核通过后再公开。
