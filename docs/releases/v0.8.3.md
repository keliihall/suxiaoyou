## P0 发布阻断项

### 0. 英文界面完整本地化与品牌契约

- 所有应用自有展示名称按语言固定为：英文 `suyo`、简体中文“苏小有”；内部包、进程、协议、存储键和 bundle identifier 继续使用 `suxiaoyou`。
- 英文翻译资源不得包含中文字符；首次引导、启动/崩溃兜底、移动端、设置、活动记录、服务端错误和原生菜单不得绕过语言层。
- 新生成的工具活动按请求语言输出；历史已持久化标题由前端兼容映射，切换语言无需重写数据库记录。
- 公开下载资产使用中性 `suyo-*` 文件名；既有 Windows、macOS 与 Linux 安装身份保持不变，以支持从 rc.1 覆盖升级，并在操作系统支持的位置提供本地化显示名。系统原生对话框的系统控件仍跟随操作系统语言。
- 公开安装包文件名统一使用 `suyo-<release>-<platform>`，但不迁移应用数据目录或已有深链接协议。

验收：以英文启动并巡航桌面首页、设置各页、移动端、活动记录和错误态时，应用自有可见文本不包含中文字符；原生菜单与托盘切换语言后同步更新；中英文品牌、菜单和资源契约均有自动化测试。

### 1. 内置 PPTX 静态预览

- 在应用内显示幻灯片缩略图和单页静态预览，支持翻页、页码和缩放。
- 优先采用可再分发、跨平台且可在安装包内离线运行的渲染链路；新增依赖必须通过许可证审计。
- 不执行文档中的宏、脚本、外部链接或嵌入可执行内容。
- 不支持的图形、字体或媒体必须显示明确降级提示，并保留下载和外部应用打开。
- 保留文件大小、页数、解压总量和渲染超时限制，防止压缩炸弹与内存失控。

验收：在 macOS、Windows 和 Linux 安装包中可离线预览包含中英文、图片、表格和基本图形的 PPTX；无法完整渲染时不崩溃、不空白，并提供降级出口。

### 2. 可选已安装应用的“打开方式”

- 文件菜单保留“使用默认应用打开”，新增“选择其他应用”。
- 使用各平台原生选择器或可验证的系统接口，不在前端自行扫描应用目录。
- 仅本地桌面会话可见；远程访问不得触发本机应用。
- 应用选择取消、目标应用失效或系统拒绝时，给出可理解的非阻断错误。

验收：三平台上可分别用默认应用和用户选定的其他应用打开同一文件；远程令牌和未授权路径无法调用本机应用。

### 3. 大文件原生“另存为”

- 文件复制在 Tauri/Rust 原生层或后端以文件路径流式完成，不再将完整内容转换为 base64 或 JavaScript 数组。
- 目标路径由原生保存对话框授权，源路径继续通过工作区/生成文件授权校验。
- 复制到同目录临时文件，`fsync` 成功后原子替换；失败不得破坏已有目标文件。
- 支持大于 50 MiB 的文件，并对磁盘空间不足、重名覆盖、权限拒绝和源文件消失给出明确结果。

RC 验收分两层：自动化门禁验证 1 MiB 分块传输、源文件句柄绑定、目标临时文件同步与原子替换、失败不破坏旧目标；三平台 1 GiB 文件的内存峰值与 SHA-256 对照列为 RC 安装包发布后的人工抽验，不把尚未执行的人工结果写成自动化已通过。

### 原生文件操作的安全边界

- 源文件由后端使用持久化的会话工作区授权，逐级拒绝符号链接和 Windows reparse point；远程请求不得触发主机原生操作。
- “另存为”和源文件流式读取绑定授权时已打开的文件句柄；即使同一用户的其他进程随后把原路径替换为符号链接，也不会读取替换后的路径字节。
- 默认打开和文件管理器定位依赖只接收文本路径的系统接口；后端会在同一同步启动入口重新打开源文件、比对不透明 identity，并持有初始与重验句柄直到启动器调用返回。
- “打开方式”在原生应用选择或应用合法性校验后，再向后端取得 canonical path 与不透明 identity 并与首次授权结果比对；第二次授权紧邻 ShellExecute/LaunchServices/gio 启动调用，但不声称跨后端 HTTP 请求与系统启动器持有同一文件句柄。
- 上一条承诺的边界是：防止远程/WebView 提供工作区外路径，并拒绝在最终启动器检查前已发生的路径替换；不声称可阻止一个已拥有同一操作系统账户和工作区写权限的恶意本地进程，在最后一条检查指令之后竞速改名路径。这不是远程路径越界能力；使用快照、硬链接或 `/proc/self/fd` 会破坏“编辑原文件/定位原路径”语义，因此本版本不采用这些替代方案。

## P1 核心体验项

### 4. 多轮对话历史定位

#### 产品形态

- 在对话内容左侧显示窄型纵向导航轨道；每一轮以用户消息为起点，对应一个刻度。
- 刻度的位置表示该轮在整体对话中的顺序，不用 DOM 像素高度直接映射，避免超长工具输出使刻度失真。
- 当前视口所在轮次使用高亮长刻度；其他轮次使用低对比度短刻度。
- 悬停或键盘聚焦刻度时，显示该轮的用户问题摘要、时间及附件名；摘要仅使用本地已存储文本，不为此额外调用模型。
- 点击或按 Enter/Space 平滑定位到用户消息顶部；连续键盘上/下可切换上一轮/下一轮。
- 轨道只在轮次达到阈值（建议 4 轮）且内容区高度充足时显示；窄屏默认折叠为“对话大纲”按钮，不挤压正文。
- 滚动到底部按钮、新消息未读数、编辑重发与流式输出的现有行为保持不变。

#### 技术方案

- 后端新增轻量轮次索引接口，返回用户消息 ID、顺序、创建时间、截断摘要和附件名，不返回完整 parts 或工具输出。
- 索引必须包含尚未进入前端 50 条分页的早期轮次，使轨道能代表完整对话。
- 点击已加载轮次时直接使用消息 ID 定位；点击未加载轮次时，按目标 offset 定向取回所在页及相邻页，挂载后再定位。
- 每个用户消息容器增加稳定的 `data-message-id` 与 ref 注册；用 `IntersectionObserver` 判定当前轮次。
- 用户主动点击导航后暂停自动跟随底部；新消息继续通过现有未读计数提示，不把视口拉回底部。
- 编辑重发导致后续消息被删除时，索引和选中状态必须立即失效，不保留幽灵刻度。

#### 验收

- 200 轮对话首屏不加载全部消息正文，但导航轨道可表示完整轮次并直接跳转至第 1、100、200 轮。
- 目标轮未加载时显示短暂的定位进度，请求失败可重试，不破坏当前滚动位置。
- 用户手动滚动、点击刻度、加载旧页和持续流式输出时，高亮轮次稳定且不出现反复跳动。
- 浅色/深色主题、200% 缩放、键盘操作和屏幕阅读器均可用；刻度热区不小于 24 px，焦点不依赖颜色表达。
- 增加后端索引/定向分页测试、前端导航状态单测试和 Playwright 长对话回归。

## 实施结果

1. 原生另存为与可选应用打开共用会话路径授权，文件正文以 1 MiB 分块从后端流入 Rust 临时文件，成功同步后原子替换目标。
2. PPTX 使用项目已锁定的 MIT `python-pptx` 生成受限静态场景模型，前端以 React/CSS/SVG 离线渲染；不新增许可依赖，不读取外部关系。
3. 历史定位使用完整轻量轮次索引和隔离的定向分页窗口，不污染实时最新页缓存；覆盖键盘、窄屏、失败重试和新消息未读状态。
4. 发布标签触发 Windows、macOS 与 Linux 原生构建矩阵，全部验证后公开 `v0.8.3-rc.2` pre-release。
5. rc.2 统一英文 `suyo` / 中文“苏小有”展示契约，补齐 Web、移动端、后端动态文本、Tauri 原生菜单与 Windows 安装层的双语链路；排队输入按条保存语言，并加入历史文案双向渲染与反向本地化门禁。
