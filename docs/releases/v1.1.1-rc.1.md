# 苏小有 v1.1.1-rc.1 发布说明

发布日期：2026-07-24

`v1.1.1-rc.1` 是应用版本 `1.1.1` 的候选测试版。它使用
`UNSIGNED-DEGRADED` 发布配置，公开为 prerelease、不会标记为 latest，也不代表已经满足正式发行所需的
签名、公证、authoritative Office renderer、真实集成或受控 Beta 证据。

## 本次修复

旧版将 POSIX `st_dev`/`st_ino` 写入数据库并作为跨重启的工作区身份。macOS/APFS 在合法重启或重新挂载
后可能改变 device 编号，即使目录没有被替换，后端也会误判 checkpoint 工作区冲突并在就绪前退出。

v1.1.1 改为 workspace identity v2：

- POSIX 普通工作区使用随机 `marker-v2` durable token，优先存入目录 xattr；文件系统不支持 xattr 时，
  使用无符号链接跟随、create-only、原子发布并持久化的安全标记文件。
- Windows 使用目录句柄取得卷序列号和 file ID，生成 `winfile-v2` token，不在用户工作区创建标记文件。
- 原生 device/inode 只保护当前操作的 TOCTOU 边界，不再充当 POSIX 跨重启的连续性证据。
- 旧 `stat-v1` 工作区先建立 durable token，再复制并完整校验文件版本树，最后才更新数据库。
  中途崩溃可幂等重试，旧历史仍会保留。
- 缺失、被替换、损坏或身份不可证的工作区、checkpoint 与 transaction journal 会分别保留并阻断；
  安全工作区继续恢复，本地服务不会因单个历史条目而整体启动失败。
- checkpoint、rewind、文件版本、实时工具事务和应用管理的 Git worktree 全部验证同一 durable token。
  删除后在相同路径创建的新目录不能继承旧历史。

## Windows ARM64 远程隧道边界

截至本候选版发布时，Cloudflare 官方 `cloudflared` Release 提供 Windows 386 与 amd64 资产，但没有
Windows ARM64 原生资产。苏小有不会把 amd64/x64 `cloudflared.exe` 自动下载到 Windows ARM64 包中并
冒充原生 ARM64 二进制。

- 如果系统 `PATH` 中没有用户自行提供且与 Windows ARM64 环境兼容的 `cloudflared.exe`，启用
  Cloudflare Quick Tunnel 会明确失败关闭：不会创建隧道，也不会放宽远程访问认证或 CSRF 允许来源。
- 该限制只影响可选的远程隧道。用户可以暂不启用隧道，继续使用本地聊天、工作区、文件与恢复功能。
- 确需隧道时，请自行提供经过验证的兼容二进制并确保应用可从 `PATH` 找到；在 Cloudflare 发布官方
  Windows ARM64 资产并完成本项目的原生验证前，本应用不会自动选择 x64 下载作为替代。

## 安装与升级

- 安装前退出正在运行的苏小有。首次启动会自动迁移已登记的旧工作区身份和文件版本历史。
- 不要手工删除工作区身份 xattr、`.suxiaoyou/workspace-identity-v2`、transaction journal 或旧文件版本树；
  它们是恢复与防止同路径误绑定所需的连续性证据。
- 被单独阻断的历史工作区不会阻止应用启动，但在其身份或历史得到人工确认前，不应对它执行 rewind。
- RC 安装包内部版本是 `1.1.1`。未来安装同内部版本的后续候选包或正式包时，部分系统可能需要先卸载
  当前候选版或显式覆盖安装。

## 发布信任边界

- Windows x64/ARM64 NSIS 均未做 Authenticode 发行方签名，SmartScreen 可能警告。
- macOS 应用仅为 ad-hoc 签名；DMG 未做 Developer ID 签名、Apple 公证或 staple，Gatekeeper 可能拦截。
- Linux DEB/RPM 未做包签名或仓库签名。
- 公开包不包含冻结 authoritative Office renderer；权威 Office authoring/视觉提交保持不可用。
- 腾讯文档、图像 Provider 等真实集成证据不包含在此降级发布中。

请使用同一 GitHub Release 中的 `CHECKSUMS.md` 和 `release-manifest.json` 校验下载文件。机器可读清单会将
上述信任与能力限制固定到 `v1.1.1-rc.1` 标签和对应的完整 Git commit。
