# 苏小有 v1.1 开发与发布合同

状态：`v1.1.0` 的 `UNSIGNED-DEGRADED` 直接发布已获产品所有者显式批准；该配置公开但标记为
prerelease、非 latest，不等同于通过本合同 GA 门禁
目标版本：`1.1.0`
发布主题：可验证、可回退的本地 Agent 与高保真 Office 闭环
范围冻结日期：2026-07-17
发布分层：GA 核心能力与显式标记的 Beta 集成能力共同进入 v1.1

## v1.0 RC 防火墙

v1.1 可以在独立开发线并行实施，但 v1.0 仍处于 RC 候选准备状态。以下规则在 v1.0 GA 前不可豁免：

1. v1.1 的数据库迁移、功能开关、依赖、renderer、ACP 入口、Hook、安装资产和版本元数据不得进入或
   重建任何 `v1.0.0-*` tag；v1.0 候选证据只能证明其绑定的 v1.0 tag/commit。
2. v1.1 测试、包和 Beta 数据使用独立的版本、应用数据目录、凭据命名空间和证据目录，不能计入
   [`v1.0 RC 验收计分卡`](v1.0-rc-scorecard.md)。
3. v1.0 修复若来自 v1.1，必须以最小补丁独立评审、独立回归并重新生成同一候选 tag 规则要求的证据；
   不得把 v1.1 功能依赖作为 v1.0 修复的前提。
4. v1.1 开发不得降低 [`v1.0 发布合同`](v1.0.0.md) 的五个原生目标、七包、签名公证、真实集成和
   20 人/200 任务/7 天 Beta 门禁，也不得用 v1.1 的 renderer 或兼容结果追认旧候选包。
5. v1.1 可以完成 M0 至 M5；只有 v1.0 已 GA，或项目以单独、公开且逐项审计的决策终止 v1.0 后，
   才能创建 v1.1 RC tag。普通排期压力不能触发后一种决策。

任何违反以上规则的合并、tag 或证据复用都立即停止 v1.1 发布工作，先恢复发布线和证据隔离。

### 2026-07-18 显式覆盖决定

产品所有者已明确要求忽略上述发布堵点并直接执行 `v1.1.0`。本决定暂停 v1.0 发布线，不复用或
追认任何 v1.0 证据；同时只授权创建带机器可读降级声明的 `UNSIGNED-DEGRADED` prerelease。
缺失的签名、公证、authoritative renderer、真实集成、corpus 与 Beta 证据继续如实记录，不能因该
覆盖决定被写成“已通过”。

## 完成定义

v1.1 完成必须同时满足以下条件：

1. v1.0 RC 防火墙持续成立，v1.1 所有证据绑定同一个 v1.1 候选 tag、commit、依赖锁、renderer/字体
   指纹和测试清单。
2. 统一运行时事件与持久 turn checkpoint 覆盖用户 turn、工具、子 Agent、Hooks、ACP、Office 提交和
   验证；文件提交成功不会早于 mutation ledger 持久化。
3. GA rewind 原子恢复苏小有受控 turn 的对话前沿和本地创建、修改、删除及 Office 多文件变更；冲突
   fail-closed；所有外部副作用均明确标记不可逆。
4. GA Hooks 在来源级权限、持久审计和急停之内运行；`PreToolUse` 可以允许、拒绝或请求确认，但不能
   改写参数或扩大权限，被拒调用产生零副作用。
5. Beta ACP 通过锁定协议版本的一致性套件和真实客户端 smoke，只开放本地 stdio；权限、取消、去重、
   checkpoint 和审计与桌面端同源，不存在 HTTP 或远程旁路。
6. Beta Git worktree 对干净仓库完成受控创建、绑定、隔离、清理和崩溃对账；不删除脏目录、不跨实例
   写入、不用 Git reset/历史改写冒充 rewind。
7. 验证 Agent 是独立只读执行者，输出结构化结论和证据，不能写文件、批准权限或完成自己的 checkpoint；
   Office 修复最多两轮，确定性失败不能被模型判为通过。
8. DOCX、XLSX、PPTX 在冻结支持矩阵内完成高保真编辑；未触及 OOXML part/relationship/media 摘要不变，
   不支持输入在提交前原样拒绝。
9. 高保真预览由固定的本地 renderer/字体指纹生成 PDF 和逐页 PNG；近似预览明确降级标记；模板、图表、
   复杂格式、版本、视觉验证和 rewind 形成同一 checkpoint 闭环。
10. Windows x64、macOS arm64/x64、Linux x64/ARM64 的同 tag 原生矩阵、七个安装包、完整回归、安全与
    供应链审计、Office corpus 和受控 Beta 通过；不存在 P0/P1、权限绕过、静默数据损坏或不可恢复覆盖。

Beta 表示 v1.1 中受功能开关、显著标签和额外确认保护的可用能力，不表示可跳过完成定义。任何 Beta
任务未达到本文件的 Beta 门禁，仍会阻断 v1.1 发布；改变范围必须先显式修订本合同，不能在 RC 时静默
移除或改名。

## 冻结范围

### GA

- 统一运行时事件、`root_turn_id`、`workspace_instance_id`、turn checkpoint、mutation ledger 与持久
  journal；
- 本地受控状态的 checkpoint 列表、预检、rewind、崩溃恢复、冲突报告与不可逆副作用提示；
- 应用内置及按内容 SHA-256 授权的项目/插件本地命令 Hooks；
- Office 本地高保真渲染、结构 delta、before/after 视觉差异、版本与 rewind 联动；
- 第一方签名模板，DOCX/XLSX/PPTX 的冻结复杂格式和图表矩阵；
- Office 专用的确定性验证与只读验证 Agent 闭环；
- Security Center 中的 checkpoint、Hook、ACP、worktree、validator 来源、能力和审计视图。

### Beta

- 仅 stdio 的标准 ACP 会话、提示、取消、消息/计划/工具更新和交互权限请求；
- 仅干净 Git 仓库的 detached worktree 生命周期、会话/子 Agent 绑定和安全清理；
- 代码/通用文件任务的只读验证 Agent 和结构化证据；
- 经过 OOXML 安全检查、placeholder contract、独立重开和渲染验证的用户模板导入。

### 明确排除

- ACP WebSocket、远程 relay、公开分享、云同步、无人工值守远程写入和自定义权限扩展；
- HTTP Hook、参数/工具名改写、秘密自动注入、动态代码加载和后台常驻 Hook；
- 自动搬运脏 Git 改动、共享分支并发 worktree、自动 merge/push、`reset --hard`、rebase 或远程历史改写；
- 对邮件、MCP、云文档、付费 API、打印、上传和 Git 远程操作宣称 rewind；
- Best-of-N 自动选优、验证 Agent 自主写入或无限修复循环；
- Office 宏、VBA、OLE/ActiveX、SmartArt 结构编辑、动画、数据透视表、外部数据连接、任意修订/批注
  保真、Excel 原生公式等价重算和任意第三方模板无损编辑；
- 依赖 Office GUI 自动化的创建/编辑、把文档上传到云端渲染，或把浏览器近似预览标为高保真；
- 多租户云 Agent Runtime、Teams/RBAC、共享项目云盘、无 PC 云执行和原生移动客户端。

## Office v1.1 支持矩阵

所有格式继续要求工作区约束、路径/扩展名/符号链接防护、输入与资源预算、写入前版本、私有草稿、独立
重开、安全检查、结构 delta、原子提交和 checkpoint。以下能力只对安全检查通过且目标结构唯一的 OOXML
文件成立。

### DOCX

- 继承或应用明确样式，编辑节、纸张、方向、边距、页眉页脚、多级列表和表格样式；
- 支持表格单元格合并、边框/底纹，图片尺寸、裁剪、对齐和分页控制；
- 使用第一方模板 placeholder 生成文档；图表以带替代文本和来源数据引用的验证后静态图像插入；
- 不承诺浮动对象在所有 Office 套件像素一致，不编辑复杂域、脚注/尾注关系或修订历史。

### XLSX

- 保留 v1 的单元格、工作表、公式写入和基础样式，增加合并单元格、行高/列宽、冻结窗格、筛选、
  条件格式、数据验证和命名范围；
- 支持以明确数据区域创建或更新柱状图、折线图、饼图和散点图；
- 公式只写入并保留缓存语义，不宣称使用 Excel 计算引擎完成重算；预览必须标明缓存值是否可用；
- 不编辑数据透视表缓存、宏、外部链接、Power Query、任意二进制对象或不支持的 chart extension。

### PPTX

- 复用第一方模板的 theme、slide master 和已有 layout 创建页面，不提供母版创作器；
- 支持文本样式、基本形状与分组、图片裁剪/对齐、表格样式、演讲者备注，以及柱状图、折线图、饼图
  和散点图；
- 未选择的页面、layout、master、theme、media 和 relationship 摘要必须保持不变；
- 不编辑动画、切换、SmartArt、宏、嵌入对象或未知扩展。

第一方模板属于 GA；用户模板属于 Beta。模板不能扩大上述支持矩阵。

## 证据门禁

### 运行时、rewind 与 worktree

- 事件 ID 唯一，单个 `root_turn_id` 的持久序列确定；重复投递不重复工具或副作用；并行 turn 不串用
  checkpoint/workspace instance。
- 在每个 journal 持久化边界注入进程终止后，受支持的本地状态最终只能是完整前态或完整后态；机器
  可读报告列出预期/实际 SHA-256、DB 前沿和恢复动作。
- rewind 覆盖创建、修改、删除、目录和至少每格式 20 个多文件 Office 案例；当前摘要冲突时所有目标、
  消息、待办、Goal 和缓存保持不变；重复请求结果相同。
- 所有不可逆外部调用在预检和结果中可见。测试不得用 mock “成功撤回”来证明外部 rewind。
- worktree 在五个原生目标用真实 Git 仓库通过创建/绑定/停止/清理/崩溃对账；跨实例写入成功数为零，
  脏或活跃 worktree 删除成功数为零。

### Hooks、ACP 与验证 Agent

- 每个 `PreToolUse deny` 用例的进程、网络、文件、版本、checkpoint 和外部 mock 调用增量均为零；`ask`
  只有用户批准后进入原权限管线。
- Hook 可执行文件、内容、参数、环境或工作目录任一绑定变化都会重新授权；日志和事件扫描不得出现
  测试秘密；必需/可选失败策略分别经过超时、异常退出和非法 JSON 验证。
- ACP 通过实现锁定版本的公开一致性测试，并至少用两个兼容客户端完成会话创建/加载、提示、工具状态、
  权限、取消和恢复；每次运行仅使用持有 lifespan 独占 lease 的 file-backed SQLite 单进程。
  `messageId` 重试不重复执行，history edit/delete/rewind 后不重放旧响应，顽固工具真正 quiescent
  之前不释放 session；8 小时 soak 后无孤儿进程、重复写入或未结束 journal。
- 验证 Agent 对全部种子确定性失败返回 `fail` 或 `needs_review`，写工具和权限批准调用成功数为零；达到
  两轮或预算后停止，不能自动开启新 root turn 绕过限制。

### Office

- GA corpus 至少 300 份：DOCX、XLSX、PPTX 各不少于 100 份；每格式包含不少于 20 份 CJK/混合字体、
  20 份模板/图表/复杂格式、20 份编辑/rewind 和 20 份恶意或明确不支持输入，其余覆盖边界组合。
- 全部支持样例在五个原生目标独立重开并生成结构报告、PDF/PNG 和 renderer/字体指纹；不支持样例提交数
  为零且源文件 SHA-256 不变。
- 冻结包中的 canonical release identity 与实际版本/同 tag commit 一致；attestation v2、bundle tree、
  dependency manifest 和直接解析的 PE/Mach-O/ELF 闭包完全一致。每个目标必须从签名包实际
  运行 144-DPI DOCX probe，并产生与签名 golden 一致的 PDF、canonical RGBA pixel 摘要和嵌入字体证据。
- 进程/环境/Fontconfig 隔离需在发布包上证明路径、字体和临时目录没有回退宿主状态；另外必须有
  macOS XPC/App Sandbox、Windows AppContainer 或 Linux namespace/seccomp/cgroup 的对抗行为报告。
- 每个 golden case 的 manifest 固定允许变化的 part/relationship、页面区域、忽略掩码和阈值；未触及
  OOXML part/media 的规范摘要必须一致。renderer 或字体指纹变化时全部相关基线失效并人工重批。
- Microsoft Office、WPS 和 LibreOffice 在其可用原生平台各抽验每格式不少于 30 份支持样例；结果按
  套件/版本披露，不能把某一套件结果外推成三者等价。
- restore/rewind 后，文件摘要等于 checkpoint 摘要，缓存键和界面版本指向恢复文件；陈旧高保真预览
  展示数为零。公式未重算、近似预览和 `needs_review` 均必须可见。

### 发布与 Beta

- 同一候选 tag 由五个原生 runner 生成 Windows x64、macOS arm64/x64、Linux x64/ARM64 的报告和七个
  安装包；完成安装、升级、启动、核心任务、退出和无孤儿进程检查。稳定 macOS 包使用 Developer ID
  签名并完成公证。
- 完整后端、前端、Rust、Playwright、迁移、恢复、打包、许可、SBOM 与供应链 vulnerability 门禁通过；
  跳过项逐项说明且不能覆盖发布功能。
- Beta 至少 30 名用户、500 个受支持核心任务、连续 14 天；其中 Office mutation、rewind、ACP、worktree、
  Hooks/validator 各不少于 50 个任务（同一复合任务可以进入多个分层）。每项成功率至少 95%，权限绕过、
  静默数据损坏和不可恢复数据丢失均为零；所有外部不可逆提示有确认记录。

## 停止、降级与恢复条件

- 发现 v1.0 RC 防火墙破坏：停止 v1.1 合并/tag/打包，撤销证据污染并重新对账；不得只改文案继续。
- checkpoint 漏记受控文件、提交成功早于 ledger 持久化，或崩溃后出现文件/对话半恢复：停止所有写功能
  扩展，先修复状态模型并重跑全套崩溃注入。
- rewind 覆盖用户或第三方进程的新内容、隐藏不可逆副作用，或通过 `git reset --hard` 实现：停止 rewind
  和 worktree，保留只读诊断，直到架构复审通过。
- Hook、ACP、子 Agent 或验证 Agent 绕过来源级权限、审计、急停、工作区边界，或泄漏凭据：立即默认
  关闭相关入口，轮换受影响凭据，完成安全事件审计后才能恢复开发。
- worktree 清理删除非应用所有、脏、活跃或被 checkpoint 引用的目录：停止自动清理；完成恢复和所有权
  对账前不得重新开启。
- Office 出现静默损坏、未触及 part 改变、不支持输入被部分提交、renderer 访问网络、陈旧预览被标为
  当前版本，或验证 Agent 自我批准：停止 Office 写入，保留原文件与 journal，先完成根因和 corpus 回归。
- 任一原生目标不能提供固定 renderer/字体和安全运行证据：该目标不能宣称高保真 GA；不能用近似预览
  静默降级。由于高保真预览是本版本 GA 核心，若无法在冻结范围内修复则延期 v1.1。
- 功能冻结后存在 P0/P1、供应链 vulnerability、权限绕过、迁移/回滚失败、不可恢复覆盖，或 Beta 未达到
  数量/周期/成功率：延期 RC/GA，不降低阈值、不把 GA 改名 Beta 规避门禁。

本合同没有 Stretch 范围。只有全部完成定义已经由同 tag 证据证明后，新增能力才可以进入后续版本。
