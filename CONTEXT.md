# 苏小有项目上下文

本文档用于说明苏小有源码中的核心概念，方便后续维护时保持命名一致。

## 产品定位

苏小有是安装在电脑端的 AI 工作助理，重点处理 Word、Excel、PPT、PDF 和本地资料，帮助用户生成报告、纪要、分析、计划、邮件等办公成果。

产品强调本地优先：文件、数据、对话和工作流程尽量留在用户自己的电脑上。

## 核心概念

**会话**

用户和苏小有围绕一个任务持续交流的上下文。侧边栏中的每条对话都对应一个会话。

**消息**

会话中的一次用户输入或助手回复。

**片段**

消息内部的最小内容单元，例如文本、工具调用、思考过程、文件卡片、生成文件或压缩摘要。

**工作区**

用户选择的本地目录。苏小有读取、写入和整理文件时，应优先围绕工作区进行。

**项目**

用于持久化工作区路径和元信息的数据记录。界面上可理解为一个工作资料范围。

**模型服务商**

提供模型能力的后端来源，可以是本地模型、本地兼容接口，也可以是用户自己配置的云端模型服务商。

**工具**

苏小有可以执行的具体能力，例如读取文件、写入文件、搜索、生成文档、运行命令等。

**权限**

控制工具是否可以执行的规则。涉及文件修改、命令执行、网络访问等操作时，应尊重用户选择。

**生成文件**

由苏小有生成并可下载或继续编辑的成果文件，例如报告、PPT、表格、网页或 Markdown 文档。

**技能**

沉淀下来的可复用流程说明，例如 macOS DMG 打包流程。技能用于让后续任务更稳定、更少依赖临时记忆。

## 命名原则

- 面向用户的文字优先使用中文。
- 产品名统一使用“苏小有”。
- 旧名称、旧品牌、旧仓库路径不应出现在用户可见界面和新文档中。
- 技术代码中的英文标识可以保留，但展示给用户的说明要尽量中文化。
